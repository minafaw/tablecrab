Ocr Editor
dialog to adjust optical schar recognition of images

this dialog is currently under construction and can not be accessed through TableCrab. you have to start it directly via >>python Tc2DialogOcrEditor.py. some future version of TableCrab may add functionality to let users adjust optical char recognition. until then the dialog should be useful as developer tool. in the background of the dialog gocr, an open source optical char recognition engine, does the dirty work.

Tool bar
Scan image
scans current image

Open image..
opens an image from disk for scanning

Save image..
saves current image to disk

Open settings..
loads settings from disk

Save settings..
saves current settigs to disk

left pane - Settings
the settings pane to the left holds various settings to be adjusted for scanning

Chars
if enabled, limits chars to be recognized to the specified subset. range notation 0-9 A-Z is supported

Gray level
if enabled, adjusts gray level for scanning. the ocr engine assumes black is text, white is background, this setting adjusts the threshold (0-255)

Dust size
if enabled, adjust size of dust (in pixels) to be filtered from image

Word spacing
if enabled, adjusts size of word spacing as opposed to char spacing (in pixels)

Ceratinty
if enabled, adjusts the certainty required to recognize a char (0-100). NOTE: unrecognized chars are represented in the output by an underscore

Invert image
if checked, inverts the image before scanning

Analyze layout
if checked, the ocr engine applies layout analysis

Context correction
if checked, the ocr engine applies context correction on chars

Compare unknown chars
if checked, the ocr engine tries to recongize unrecognized chars by comparing them to recognized ones

Divide overlapping chars
if checked, the ocr engine tries to divide overlappng chars

Pack chars
if checked, the ocr engine does ..well ..whatever ocr engines do when packing chars

Output type
if enabled, tries to convert the output to the specified type ..integer or floating point number are supported types for now.

Output pattern
if enabled, filters the output through the specified regular expression pattern. i assume you are familliar with pythons regex engine. extended, multiline patterns are ok. unicode flag will be set. the pattern should contain a named group "output" that matches whatever is is intended to be returned as output. filtering is applied before type conversion is done (if any).

top pane - Image
holds the image to be scanned
bottom pane - Output
output is dumped to the left sub pane, errors to the right one.
